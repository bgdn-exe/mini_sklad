# Мини-Склад на 1С

Учебный pet project на платформе 1С:Предприятие 8.3.

Цель проекта — поэтапно разобраться с базовыми объектами 1С: справочниками, документами, табличными частями, общими модулями, регистрами накопления, проведением документов, отчётами и выгрузкой конфигурации в GitHub.

## Что реализовано

### Версия v0.1

Реализована базовая структура конфигурации:

- подсистема `Мини-Склад`;
- справочник `Товары`;
- справочник `Контрагенты`;
- документ `Поступление`;
- документ `Реализация`;
- табличные части документов с полями:
  - товар;
  - количество;
  - цена;
  - сумма;
- автоматический расчёт суммы строки;
- общий модуль `РаботаСДоками`;
- обработчики формы при изменении количества и цены.

### Версия v0.2

Добавлена базовая складская логика:

- справочник `Склады`;
- регистр накопления `ОстаткиТоваров`;
- измерения регистра:
  - `Склад`;
  - `Товар`;
- ресурс регистра:
  - `Количество`;
- документ `Поступление` делает движение `Приход`;
- документ `Реализация` делает движение `Расход`;
- добавлен отчёт `Остатки по товарам`.

Теперь система умеет учитывать остатки товаров по складам.

## Основные объекты конфигурации

### Справочники

`Товары` — хранит список товаров.

`Контрагенты` — хранит поставщиков и покупателей.

`Склады` — хранит список складов, по которым ведётся учёт остатков.

### Документы

`Поступление` — фиксирует поступление товаров на склад.

`Реализация` — фиксирует продажу товаров со склада.

В документах используется табличная часть `Товары`, где указываются товар, количество, цена и сумма.

### Регистр накопления

`ОстаткиТоваров` — регистр накопления вида `Остатки`.

Он хранит движения товаров по складам:

- поступление увеличивает остаток;
- реализация уменьшает остаток.

Структура регистра:

- измерение `Склад`;
- измерение `Товар`;
- ресурс `Количество`.

### Отчёты

`Остатки по товарам` — показывает текущие остатки товаров по складам.

Отчёт построен через СКД и использует виртуальную таблицу регистра накопления:

```bsl
РегистрНакопления.ОстаткиТоваров.Остатки
```

Из неё выбираются склад, товар и остаток количества.

## Пример логики работы

Поступление:

```text
Основной склад | Молоко | 10
```

Реализация:

```text
Основной склад | Молоко | 2
```

Отчёт по остаткам покажет:

```text
Основной склад | Молоко | 8
```

## Как развернуть проект

1. Скачать репозиторий:

```bash
git clone https://github.com/bgdn-exe/mini_sklad.git
```

2. Создать пустую информационную базу 1С без конфигурации.

3. Открыть базу в режиме конфигуратора.

4. Загрузить конфигурацию из файлов:

```text
Конфигурация → Загрузить конфигурацию из файлов
```

5. Выбрать папку:

```text
src
```

6. Обновить конфигурацию базы данных:

```text
Конфигурация → Обновить конфигурацию базы данных
```

или нажать `F7`.

7. Запустить 1С:Предприятие.

8. Создать тестовые данные:

- склад;
- товары;
- контрагентов;
- документ поступления;
- документ реализации.

9. Открыть отчёт `Остатки по товарам` и нажать `Сформировать`.
