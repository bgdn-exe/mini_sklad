# Мини-Склад на 1С

Учебный pet project на платформе **1С:Предприятие 8.3** для базового складского учёта.

Проект реализует простую конфигурацию, в которой можно вести список товаров и контрагентов, оформлять поступление и реализацию товаров, а также автоматически рассчитывать сумму строк документа.

## Цель проекта

Цель проекта — изучить базовые механизмы разработки на платформе 1С:

* создание справочников;
* создание документов;
* работа с табличными частями;
* настройка форм;
* обработчики событий формы;
* использование общих модулей;
* выгрузка конфигурации в файлы;
* ведение проекта через Git.

## Реализовано

* подсистема `Мини-Склад`;
* справочник `Товары`;
* справочник `Контрагенты`;
* документ `Поступление товаров`;
* документ `Реализация товаров`;
* табличные части документов для списка товаров;
* автоматический расчёт суммы строки по формуле:

```text
Сумма = Количество × Цена
```

* общий модуль `РаботаСДоками` для расчёта суммы;
* формы документов с обработчиками изменения количества и цены.

## Основные объекты конфигурации

### Справочник `Товары`

Используется для хранения списка товаров.

Реквизиты:

* `Артикул`;
* `ЕдиницаИзмерения`.

### Справочник `Контрагенты`

Используется для хранения поставщиков и покупателей.

Реквизиты:

* `ПолноеНаименование`;
* `ИНН`;
* `Телефон`.

### Документ `Поступление товаров`

Фиксирует поступление товаров от контрагента.

Содержит:

* реквизит `Контрагент`;
* табличную часть `Товары`;
* поля `Товар`, `Количество`, `Цена`, `Сумма`.

### Документ `Реализация товаров`

Фиксирует продажу товаров контрагенту.

Содержит:

* реквизит `Контрагент`;
* табличную часть `Товары`;
* поля `Товар`, `Количество`, `Цена`, `Сумма`.

## Расчёт суммы

Расчёт суммы вынесен в общий модуль `РаботаСДоками`.

```bsl
Процедура РассчитатьСумму(СтрокаТабличнойЧасти) Экспорт

	СтрокаТабличнойЧасти.Сумма =
		СтрокаТабличнойЧасти.Количество * СтрокаТабличнойЧасти.Цена;

КонецПроцедуры
```

Процедура вызывается из форм документов при изменении количества или цены.

## Как развернуть проект

1. Создать пустую информационную базу 1С.
2. Открыть базу в режиме `Конфигуратор`.
3. Выполнить команду `Конфигурация → Загрузить конфигурацию из файлов`.
4. Указать папку `src`.
5. Обновить конфигурацию базы данных.
6. Запустить режим `1С:Предприятие`.
